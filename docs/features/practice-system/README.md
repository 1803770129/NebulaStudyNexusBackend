# 刷题系统增强文档包

## 文档目录

1. [`01-PRD-ALL-LEVELS.md`](./01-PRD-ALL-LEVELS.md)  
   全等级产品需求文档（P0/P1/P2/P3），含目标、范围、验收标准和指标。

2. [`02-PRODUCT-DELIVERY.md`](./02-PRODUCT-DELIVERY.md)  
   产品落地文档，含分阶段上线计划、灰度策略、运营方案、风险与预案。

3. [`03-TECH-ARCHITECTURE.md`](./03-TECH-ARCHITECTURE.md)  
   技术方案与架构文档，含目标架构、数据模型、API、服务流程、测试与运维方案。

4. [`04-TASK-BACKLOG.md`](./04-TASK-BACKLOG.md)  
   具体任务清单文档，按迭代和优先级拆解到后端、前端、测试、运维可执行任务。

5. [`05-DETAILED-IMPLEMENTATION-PLAN.md`](./05-DETAILED-IMPLEMENTATION-PLAN.md)  
   前后端按优先级开发的详细执行计划，含周级排期、协作节奏、Gate、风险和本周启动清单。

6. [`06-FEATURE-DETAILED-DESIGN.md`](./06-FEATURE-DETAILED-DESIGN.md)  
   按每个功能逐项展开的详细设计，包含流程、状态机、数据、API、页面、埋点和测试点。

7. [`07-IMPLEMENTATION-STUDY-GUIDE.md`](./07-IMPLEMENTATION-STUDY-GUIDE.md)  
   截至当前的开发说明与教学文档，包含已完成功能、关键代码路径、数据流、接口、测试验证与后续开发入口。

## 使用方式

1. 先按 `01-PRD` 锁定需求边界与验收口径。  
2. 再按 `03-TECH-ARCHITECTURE` 确认数据结构、接口和模块划分。  
3. 按 `04-TASK-BACKLOG` 执行开发。  
4. 上线前对照 `02-PRODUCT-DELIVERY` 完成灰度、回滚、运营与监控准备。
